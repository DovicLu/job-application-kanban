# 产品设计说明书

## 项目名称
求职申请管理看板（Job Application Kanban）

## 设计背景

### 目标用户
大学应届生（本科/硕士），主要在秋招（9-11月）或春招（3-5月）期间同时投递多家公司。

### 用户研究发现
通过对 20 名大四学生的访谈，归纳出以下核心痛点：

| 优先级 | 痛点 | 频率 |
|--------|------|------|
| P0 | 多个截止日期混乱，经常漏掉 | 18/20 |
| P0 | 不清楚每家公司当前到了哪个阶段 | 17/20 |
| P1 | 材料准备进度不清楚 | 15/20 |
| P1 | 面试安排分散在各 App，容易忘 | 14/20 |
| P2 | 不知道该阶段该做什么准备 | 12/20 |
| P2 | 投递数量多，整体进度没有感知 | 11/20 |

## 产品设计

### 核心概念：看板（Kanban）

选择看板视图而非表格/列表的原因：
- **空间感知**：用户能在 3 秒内判断各阶段的"拥堵"情况
- **心理模型匹配**：招聘本身就是一个线性流程，看板天然契合
- **拖拽扩展性**：未来版本可支持卡片拖拽移动阶段

### 信息架构

```
看板根页面
├── 统计总览（4个数字卡）
│   ├── 投递总数
│   ├── 紧急截止数（≤7天）
│   ├── 面试安排数
│   └── 已获 Offer 数
├── 紧急告警横幅（条件显示）
├── 筛选 & 搜索工具栏
└── 5 列看板
    ├── 待投递
    ├── 已投递
    ├── 笔试/测评
    ├── 面试中
    └── 结果
```

### 申请卡片设计

每张卡片包含以下信息层级：

```
[公司名称]         ← 最大字号，最高权重
[职位名称]         ← 次级信息
[类型标签] [截止标签]  ← badge 行，可扫视
[进度条]           ← 材料完成度，视觉化
[清单]             ← 可展开的细节
[面试安排]         ← 条件显示，有蓝点标记
```

### 颜色系统

采用语义化配色，避免纯装饰性用色：

| 颜色 | 语义 | 使用场景 |
|------|------|---------|
| 蓝色 #185FA5 | 信息 / 进行中 | 职位类型标签、面试标记、已投递列 |
| 橙色 #BA7517 | 警告 / 注意 | 截止日期 4-7 天 |
| 红色 #A32D2D | 紧急 | 截止日期 ≤3 天、顶部告警 |
| 绿色 #3B6D11 | 成功 / 完成 | 材料全完成、Offer 标签 |
| 灰色 #B4B2A9 | 中性 | 待投递状态 |
| 紫色 #7F77DD | 面试中 | 面试中列标识 |

### 交互设计

**点击卡片 → AI 建议面板**
- 从右侧滑入，不打断当前视图
- 显示 Loading 状态，调用 Claude API 获取建议
- 建议聚焦当前阶段的行动项

**新增申请弹窗**
- 最小化表单字段（公司、职位、类型、截止日、阶段）
- 即填即用，30秒内完成录入

**筛选 & 搜索**
- 按职位类型一键筛选（不使用下拉，减少操作步骤）
- 全文搜索支持公司名和职位名

## 技术实现

### 前端架构
- 纯原生 JavaScript，无框架依赖
- CSS Variables 实现主题系统（自动适配暗色模式）
- 响应式布局（Grid + Media Query）

### AI 集成
- 使用 Anthropic Claude API（/v1/messages）
- 模型：claude-sonnet-4-20250514
- 每次点击卡片触发一次 API 调用
- 携带完整申请上下文（公司、阶段、截止、进度）

### 数据存储
- v1.0：内存存储（页面刷新重置，适合演示）
- v1.1（规划）：LocalStorage 持久化
